# トナメ中盤攻略マスター — 5〜30BB GTOトーナメント・トレーナー

ポーカートーナメント(MTT)の中盤戦からスタートし、**優勝するまで**プレイするGTO学習アプリ。
フルハンド(リバー・ショーダウンまで)を実戦形式でプレイし、すべての判断がGTO基準で採点されます。

**プレイはこちら: https://tkchemicalwarehouse.github.io/poker-gto-trainer/**

## 起動方法

上のURLをブラウザで開くか、`index.html` をダブルクリック。インストール不要。成績はブラウザ(localStorage)に保存されます。
ホーム画面で **📱スマホ / 🖥タブレット・PC** のレイアウトと**参加人数(18/27/45人)**を選択できます。

## トーナメントの流れ

1. あなたの卓は9人。残りの参加者は他のテーブルでプレイ中(脱落が進行する)
2. 同卓の誰かが飛ぶと残り人数が減少。補欠がいる間は新しい選手が移動してくる
3. **残り9人でファイナルテーブル!** 以降は席の補充なし、卓が短くなっていく(ショートハンド/ヘッズアップ対応)
4. **最後の1人になったら優勝🏆** — トロフィー演出+ファンファーレ
5. バストしたら順位と敗因(ミス or 分散)を表示

## ルール設定

| 項目 | 設定 |
|---|---|
| ブラインド | 2,000/4,000 スタート、**2周(18ハンド)ごとにアップ** |
| アンティ | BBアンティ = BB額 |
| スタック | 全員 5〜30BB のランダム(自分も)。増減はすべて実戦結果 |
| スタック表示 | 色分けゲージ: 赤<10BB / 黄10-20 / 緑20-35 / 青35+ (卓上のチップ画像はベットとポットのみ) |
| ベットサイズ | ソルバー風抽象化: プリフロップ 2.2BB or オールイン / ポストフロップ 33%・66%・オールイン |
| 効果音 | WebAudio合成(🔊ボタンでミュート可)。優勝ファンファーレ付き |

## モード

- **トーナメント**: 優勝かバストまでプレイ。バスト時に「ミス由来か、分散か」+順位を表示
- **GTOシミュレーション**: GTOボットに自動プレイさせ、優勝率と生存分布をヒストグラム表示(「完璧に打ってもこの優勝率」の体感用)
- **成績**: 挑戦回数・優勝回数・GTO一致率・ブランダー数・分散バスト率など

## 採点

| 判定 | 意味 |
|---|---|
| ✓ GTO通り | 最高頻度のアクション |
| ✓ OK(混合戦略) | GTOが混ぜる選択肢のひとつ |
| △ 僅かなミス | EV損失小(レンジ境界近辺など) |
| ✗ ブランダー | 明確なEV損失。理由とレンジ・実数を表示 |

オールイン関連の判断は、相手レンジに対する**モンテカルロ・エクイティ vs 必要勝率**でEVを実数計算して採点します。

## GTOの中身 — GTO Wizard等との正直な比較

本アプリの「GTO」がどこまで本物かを誇張なしに記載します。

- **◎ オールイン判断(コール側)** — 相手レンジへのリアルタイム・モンテカルロエクイティ vs 必要勝率。ソルバーの計算と数学的に同等(ChipEV範囲)
- **○ プリフロップレンジ** — HRC/GTO Wizard系ナッシュチャートの**手製近似**(`js/ranges.js`)。方向性は正しいが境界ハンドで数%ズレうる。混合頻度は未実装
- **△ ポストフロップ** — 完全GTOでは**ない**。ソルバー傾向(レンジベット・SPR基準コミット・セミブラフ選択)を模したヒューリスティック+実エクイティ計算。実ソルバー解とは頻度・サイズに差異あり
- **✗ ICM** — 未実装(ChipEVのみ)。ファイナルテーブル終盤の判定は参考値

| 項目 | 本アプリ | GTO Wizard / DTO等 |
|---|---|---|
| プリフロップ解 | ナッシュ近似(手製) | ソルバーで解いた本物 |
| 混合頻度 | なし | あり |
| ポストフロップ | 傾向ヒューリスティック | 全ボード実ソルバー解 |
| ICM / FT対応 | なし | あり |
| 優勝までの通し実戦 | **あり◎** | なし(スポット訓練) |
| 分散の体感・敗因分解 | **あり◎(独自)** | なし |
| 日本語コーチ解説 | **あり◎** | 弱い |

本アプリの価値は「正解表の正確さ」ではなく、**優勝かバストまでの通し実戦・全判断の即時採点・分散を体で覚える学習体験**にあります。今後、真のナッシュ均衡計算(ショートスタック領域)・簡易ICM・混合頻度の導入を予定。
