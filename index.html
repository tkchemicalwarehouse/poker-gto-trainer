このアプリの目的

MTT中盤戦(全員5〜30BB)からスタートし、優勝するまで実戦形式でプレイします。すべての判断はGTO基準で採点され、ズレた瞬間に「なぜ間違いか」をEV・エクイティの実数で解説します。

もう一つの大事な目的: GTO通りにプレイしても、トーナメントでは頻繁にバストします。それが「ミス」なのか「分散」なのかを区別する感覚を、回数を重ねて体に入れることです。バスト時には敗因の分解と順位を表示します。

トーナメントの流れ(優勝条件)

  • 参加人数を選択(18 / 27 / 45人)。あなたの卓は9人、残りは他のテーブルでプレイ中
  • 誰かが飛ぶと残り人数が減少。補欠がいる間は卓に新しい選手が移動してくる
  • 残り10人がバブル(あと1人で入賞)
  • 残り9人でファイナルテーブル=全員イン・ザ・マネー(入賞)! 以降は席の補充なし、卓がどんどん短くなる。賞金は9位から配分
  • 最後の1人になったら優勝!🏆
  • 自分の開始スタックはランダム(5〜30BB)か、ホーム画面で固定値を選択可能

ルール設定

  • ブラインド 2,000/4,000 スタート、2周(18ハンド)ごとにアップ
  • アンティはBBアンティのみ: BBの人がBBもう1個分を追加で払う(他の人はアンティなし)
  • チップは最小1,000。すべての金額が1,000単位
  • 全員5〜30BBのランダムスタックで開始
  • クイックボタン(2.2BBレイズ / 33% / 66% / オールイン)に加え、🎚レイズ額指定スライダーでミニレイズから「1,000残しセミオールイン」まで自由に選択可能。GTO採点は最も近い標準サイズとして評価
  • スタックゲージの色: 赤=10BB未満(危険) / 黄=10〜20BB / 緑=20〜35BB / 青=35BB超

チップの額面

GTOの中身 — GTO Wizard等との正直な比較

本アプリの「GTO」がどこまで本物かを、誇張なしに記載します。

  • ◎ オープンジャム判断(13.5BB以下) — 手書きチャートではなく、自前計算したナッシュ均衡です。169×169の全ハンド対戦エクイティ表(各1.2万回シミュレーション)を元に、ポジション×スタック0.5BB刻みでフィクティシャスプレイにより均衡を解いています(ファーストコーラー近似)。「このハンドは◯BB以下でジャム」という閾値で判定します
  • ◎ オールイン判断(コール側) — 相手レンジに対するエクイティを事前計算テーブルで厳密に計算し、必要勝率と比較。表示される「エクイティ◯% / 必要勝率◯% / EV◯BB」は実数で、毎回ブレません
  • ◎ リジャム(オープンへの3ベットオールイン) — オープナーのレンジを固定し、ジャム側とコール側を均衡化した計算済み閾値(オープナーのクラス×自分の位置×有効スタック0.5BB刻み)。「このハンドは有効◯BB以下でリジャム」で判定。ヘッズアップは専用の広い均衡
  • △ 簡易ICM(賞金圧力) — ファイナルテーブルのオールイン・コール判断にMalmuth-Harville ICMによる必要勝率の上方補正を適用(全員のスタックが既知のFTのみ)。ジャム側のレンジ調整・FT前のバブル圧力は未適用なので、その部分は参考値
  • ○ オープンレイズ/BBディフェンス(コール側)のレンジ表 — GTO系チャートの傾向を再現した近似。混合頻度は境界域(±0.5BB)のみ
  • △ ポストフロップ — 完全なGTOではありません。ソルバーの傾向(ドライボードのレンジベット、SPR基準のコミット、セミブラフ選択など)を模したヒューリスティック+実エクイティ計算による近似です
